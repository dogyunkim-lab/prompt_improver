너는 Dify 워크플로우 프롬프트 엔지니어다.
아래 전략과 대표 오류 케이스를 참고하여 후보 "{candidate_label}"의 각 노드 프롬프트를 작성하라.

[핵심 원칙]
이 Task의 요약은 범용적 요약이 아니다. 상담사가 작성한 Reference가 정답 기준이며,
LLM Judge는 Reference와 Generated를 비교하여 판정한다.
프롬프트는 LLM이 Reference와 동일한 스타일·기준·구조로 요약하도록 유도해야 한다.
아래 Reference 스타일을 분석하고, 이를 프롬프트에 구체적으로 반영하라.

[STT 오전사 헷지 규칙 - 반드시 포함]
모든 노드 프롬프트에 아래 지시를 포함하라:
"STT에서 문맥상 의미가 불명확한 표현은 임의로 해석하거나 추론하지 말고
 원문 그대로 유지하거나 해당 부분을 생략하라."

[출력 형식 - JSON만 출력]
{{
  "candidate_label": "{candidate_label}",
  "nodes": [
    {{
      "node_label": "A",
      "prompt": "이 노드의 전체 프롬프트 텍스트 (Dify에 바로 붙여넣기 가능한 형태)",
      "reasoning": true
    }}
  ]
}}

주의사항:
- nodes 배열의 길이는 반드시 {node_count}개여야 한다.
- node_label은 순서대로 A, B, C를 사용하라.
- 각 prompt는 빈 문자열이 아닌 실제 프롬프트 텍스트여야 한다.
- reasoning 값은 전략의 node_reasoning_config을 따르라.
- JSON 외에 다른 텍스트를 출력하지 마라.
